HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML provides the structure and framework for our webpage.

CSS

  • CSS is the language we use to style an HTML document.
  • CSS describes how HTML elements should be displayed.
  • External style sheets are the most common way to style a webpage.
  • External style sheets allow developers to store their CSS rules in a separate file.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A box shadow property adds shadow effects around an elements frame
  • Colour can be added using semantic phrases or by using hexadecimal code

Git

  • git status allows us to check what branch we are on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • In order to sync the GitHub repository with the current working branch we need to do the following:
  • git add -A
  • git commit -m "comment"
  • git pull origin main
  • git push origin feature/branch name

JavaScript

  • Javascript enables us to add functionality to webpages.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions are the building blocks of JavaScript.
  • Functions allow us to control where a task is executed within the control flow, if at all.
  • When we create a function, we are defining a block of code that performs a specific task and giving it a unique name.
  • To execute a function we must CALL the function in our code.
  • Having to call a function makes our code more reusable and we can call it as many times as needed.
